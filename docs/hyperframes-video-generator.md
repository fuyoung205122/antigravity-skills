# HyperFrames 高級感口播影片製作流 (hyperframes-video-generator)

# 功能說明

利用 Codex 與 HyperFrames 框架，將單調的口播影片或音訊自動轉化為具備高級設計感、精確對時的動畫分鏡影片。

# 解決什麼問題

解決了傳統口播影片視覺效果單一（僅有人臉）、後期剪輯耗時費力、以及手動將視覺元素與對白精確對齊時極度繁瑣的問題。它透過自動化流程，讓非專業剪輯師也能產出具備網頁級設計感的動態影像。

# 適合什麼情境

1. **知識型創作者**：將艱澀的教學口播配上直觀的動畫圖表。
2. **產品介紹**：快速將產品文案轉化為具備現代網頁感的展示影片。
3. **社群媒體短片**：製作風格統一、節奏感強的個人品牌宣傳片。
4. **Podcast 轉影片**：為純音訊加上豐富的視覺分鏡，提升社群傳播力。
5. **企業簡報動畫化**：將枯燥的報告內容轉化為高質感的視覺呈現。

# 如何觸發

範例 1：
「我有一段口播影片，請幫我啟動 HyperFrames 工作流，將它轉換成高級感動畫。」

範例 2：
「請使用 hyperframes-video-generator 處理這份音檔，我需要先生成視覺設計資產。」

範例 3：
「啟動 HyperFrames 流程：根據這份 SRT 字幕與我的設計草圖，產出 storyboard.md。」

範例 4：
「我想製作一個 HyperFrames 影片，請引導我完成從設計規範到代碼渲染的步驟。」

範例 5：
「執行 hyperframes-video-generator，將此產品說明音訊轉化為精確對時的分鏡代碼。」

# 輸入內容

1. **視覺設計指令**：風格提示詞或參考圖（用於生成 `design.md`）。
2. **原始素材**：口播影片檔或純音訊檔（用於轉錄字幕）。
3. **環境參數**：已安裝 Node.js、FFmpeg 與 Chrome 的本地端環境。

# 輸出內容

1. **design.md**：定義影片視覺風格與排版的設計說明書。
2. **.srt 檔案**：帶有精確時間軸的轉錄腳本。
3. **storyboard.md**：包含畫面佈局、分鏡節奏與內容說明的詳細故事版。
4. **HyperFrames 代碼**：可供渲染的 HTML/CSS/JS 代碼。
5. **MP4 影片**：最終生成的高級感動畫影片。

# 注意事項

*   **環境依賴**：本地端必須正確配置 FFmpeg 與 Node.js 才能順利執行渲染。
*   **人工校對**：在「步驟 3：創建故事版」後，建議人工檢視分鏡內容是否符合預期，再進行代碼生成。
*   **設計限制**：影片的複雜度受限於 HyperFrames 框架所支援的動畫效果與 Agent 的編程能力。

# 常見問題

**Q：如果我沒有視覺設計基礎，如何生成 `design.md`？**
A：可以利用內建的 Claude design 或 Open design 工具，輸入簡單的風格描述（如「極簡科技感」或「馬卡龍配色」），Agent 會自動為你轉化為設計規範。

**Q：字幕與畫面對不上怎麼辦？**
A：請確保在步驟 2 使用 OpenAI Whisper 生成帶有精確微秒級時間軸的 `.srt` 檔，這是後續 Agent 進行對時渲染的關鍵依據。

**Q：渲染出來的影片畫質可以調整嗎？**
A：可以，在 HyperFrames Studio 渲染階段，可透過調整 Chrome 渲染參數或 FFmpeg 指令來設定解析度與幀率。
