# Skill 名稱

內部溝通協助 (internal-comms)

# 功能說明

一套資源，幫助我撰寫各種內部溝通文件，並遵循公司偏好的格式與內容要求。

# 解決什麼問題

確保內部溝通文件（如狀態報告、領導層更新、公司通訊等）能遵循公司規範的格式和內容要求，提升撰寫效率、內容一致性與專業度。避免因格式不統一或內容遺漏而造成溝通障礙。

# 適合什麼情境

*   撰寫 3P 更新 (Progress, Plans, Problems)
*   撰寫公司內部通訊
*   撰寫常見問題 (FAQ) 回覆
*   撰寫團隊或專案狀態報告
*   撰寫給領導層的更新報告
*   撰寫專案進度更新
*   撰寫事件報告
*   需要統一格式撰寫任何內部訊息時。

# 如何觸發

範例1：
"請幫我寫一份關於新專案進度的狀態報告。"

範例2：
"我需要一份發給全公司的年度通訊稿，主題是公司成就與未來展望。"

範例3：
"請起草一份關於某次系統中斷的事件報告，格式要符合內部規範。"

範例4：
"為我們產品的常見問題集 (FAQ) 撰寫一份標準化的回覆，關於如何重設密碼。"

範例5：
"請根據 3P 格式，撰寫一份我的團隊上週進度更新。"

範例6：
"我需要一份給高階主管的專案里程碑達成更新。"

# 輸入內容

*   **明確指定溝通文件類型**：例如「狀態報告」、「公司通訊」、「FAQ」、「事件報告」等。
*   **提供核心內容資訊**：根據文件類型，提供必要的細節、數據、進度、問題點、解決方案、問答內容等。
*   （可選）**說明寫作目的與受眾**：提供這些資訊能讓 Skill 更好地調整語氣和內容，產出更精確的文件。

# 輸出內容

*   一份結構化且格式化的內部溝通文件，內容將根據您指定的類型和提供的資訊生成。
*   文件將遵循公司預設的格式、語氣和內容要求。
*   例如：一篇已排版的 3P 更新報告、一份公司通訊草稿、一份標準化的 FAQ 回覆等。

# 注意事項

*   **需明確指定文件類型**：如果請求中未能明確指出溝通文件類型，或提供的類型不符合現有指南，Skill 可能會要求進一步澄清或提供更多上下文資訊。
*   **內容需由使用者提供**：Skill 負責格式化與結構化，核心事實與細節仍需由使用者輸入。
*   **非所有類型皆有特定範本**：對於沒有明確對應特定範本的溝通類型（例如，通用內部溝通），Skill 會嘗試使用通用指南來撰寫，但可能需要更多細節指令。

# 常見問題

*   **Q: 如果我不知道要用哪種文件類型怎麼辦？**
    *   A: 請盡量提供您的目的、受眾以及希望傳達的內容，Skill 會嘗試判斷最合適的類型。如果無法判斷，會請求您提供更多資訊。
*   **Q: Skill 能自動從其他資料來源提取內容嗎？**
    *   A: 目前不能。您需要手動提供所有撰寫內部溝通文件所需的文字或數據。
*   **Q: 如果公司有新的內部溝通格式，可以更新這個 Skill 嗎？**
    *   A: 可以，但這需要 Skill 的維護者手動進行更新和配置，以納入新的指南檔案。
