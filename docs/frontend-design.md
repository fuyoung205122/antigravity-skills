# Skill 名稱

前端介面設計 (Frontend Design Skill)

# 功能說明

本 Skill 旨在建立具獨特風格、高品質且可投入生產的前端介面，專注於避免常見的「AI 生成」美學，確保產出具有鮮明設計個性的網頁 UI。

# 解決什麼問題

現今許多 AI 生成的設計常流於通用、缺乏特色，甚至帶有「AI 痕跡」。此 Skill 的存在是為了突破這些限制，專注於產出具有高設計品質、視覺衝擊力，且可實際投入生產的前端程式碼，讓您的網頁產品在眾多同質化設計中脫穎而出，展現獨特的品牌或個人風格。

# 適合什麼情境

*   需要設計開發獨特且可投入生產的網頁元件 (Web Components)。
*   希望建立具視覺吸引力的網站、登陸頁面、儀表板或完整的網頁應用程式。
*   需要撰寫高品質的 React 元件、HTML/CSS 佈局或 Vue 元件。
*   對任何現有網頁 UI 進行美化、風格調整或升級，以擺脫通用設計感。
*   追求創新、非傳統，並希望避免任何「AI 痕跡」的前端設計。
*   當您想為專案注入大膽、明確且具記憶點的設計美學。

# 如何觸發

提供至少 5 個實際範例。

範例1：
「我需要一個產品展示頁面，風格要復古未來主義，主色調使用低飽和度的藍色和紫色，並加入一些微動效。」

範例2：
「請幫我設計一個React元件，用於顯示用戶個人資料。要求極簡主義風格，注重排版和留白，字體選擇非襯線、有設計感的字體。」

範例3：
「請為一個新的SaaS儀表板設計一個登入頁面。風格偏向工業風，使用深色背景和粗獷字體，需要強調安全性和專業感。」

範例4：
「開發一個響應式的HTML/CSS佈局，用於一個藝術畫廊網站的首頁。設計要大膽、不對稱，並強調圖片的視覺衝擊力，避免網格化的常規佈局。」

範例5：
「幫我美化現有的電子商務網站導航欄，要求融入自然有機的設計元素，顏色使用大地色系，並加入流暢的Hover動畫效果。」

範例6：
「我想製作一個關於我個人作品集的單頁網站。請以『編輯雜誌』風格設計，排版需要創新且有藝術感，並搭配清晰的黑白配色方案。」

# 輸入內容

您需要提供以下資訊，以幫助 Skill 更好地理解並生成符合預期的內容：

*   **介面類型**: 您希望建立的是網頁元件、單個頁面、應用程式的某個部分，還是完整的應用程式？
*   **具體需求**: 介面的功能、包含的元素、文字內容或資料結構。
*   **目的與受眾**: 這個介面是為了解決什麼問題？目標用戶是誰？希望傳達什麼情感或訊息？
*   **技術偏好**: 偏好使用的前端框架或技術（例如：HTML/CSS/JS、React、Vue），以及是否有特定的性能或無障礙要求。
*   **美學方向**: 對於設計風格的具體描述，例如：
    *   **風格類型**: 極簡主義、奢華、復古未來主義、有機自然、編輯雜誌風、粗獷主義、Art Deco、柔軟粉彩、工業風等。
    *   **視覺元素偏好**: 顏色、字體、動效、背景紋理、空間佈局、排版等。
    *   **獨特記憶點**: 有沒有什麼是您希望用戶一眼難忘的？

# 輸出內容

本 Skill 會根據您的輸入，提供：

*   **生產級前端程式碼**: 包含 HTML、CSS 和 JavaScript（或相應的框架程式碼，如 React/Vue 元件），確保程式碼是可執行且高質量。
*   **獨特視覺設計**: 程式碼將體現您指定的或由 Skill 創造的獨特美學風格，包含：
    *   **精選字體**: 美觀、獨特且富有表現力的字體搭配。
    *   **協調配色**: 有重點、具氛圍感的色彩方案，通常使用 CSS 變數實現一致性。
    *   **高質感動效**: 具影響力且能提升用戶體驗的動畫和微互動。
    *   **創新佈局**: 不對稱、網格打破、對角流動等，具視覺衝擊力的空間構圖。
    *   **豐富細節**: 透過漸變網格、噪點紋理、幾何圖案、陰影、邊框等創造深度和氛圍。
*   **設計理念說明**: 可能會包含對設計決策、美學方向和創意選擇的闡述。
*   **精緻細節處理**: 無論是極繁還是極簡設計，都會注重每個細節的精確度與完成度。

# 注意事項

*   **明確意圖是關鍵**: 最好的結果來自於清晰且具體的設計意圖。請大膽描述您的願景，即使是抽象的形容詞也有助於引導設計方向。
*   **避免通用 AI 美學**: 本 Skill 會主動避免生成常見的 AI 設計模式，例如：過度使用的字體（Inter, Roboto, Arial）、陳腔濫調的配色（特別是白色背景上的紫色漸變）、可預測的佈局和組件模式。
*   **設計的獨特性**: 每次生成的設計都將力求獨一無二，Skill 會根據上下文進行創意詮釋和意外選擇，不會產生重複或趨同的設計方案。
*   **複雜度匹配**: 程式碼的複雜度將與設計美學願景匹配。極繁主義設計會包含豐富的動畫和效果，而極簡主義設計則注重精準、細膩的排版與細節。
*   **輸出形式**: 主要輸出為可工作的程式碼和設計概念說明，而非靜態設計稿或圖檔。

# 常見問題

**Q1: 這個 Skill 會生成設計稿或圖片嗎？**
A1: 不會，此 Skill 主要生成生產級的前端程式碼（HTML/CSS/JS, React, Vue 等）及闡述設計理念，幫助您直接進入開發階段。

**Q2: 我可以指定使用某個特定的前端框架或函式庫嗎？**
A2: 可以，您可以在需求中指定偏好的框架（例如 React, Vue）或函式庫（例如 Motion），Skill 會盡力配合並基於此進行設計與實作。

**Q3: 如果我對設計風格沒有明確想法怎麼辦？**
A3: 您可以提供介面的目的、目標受眾或想傳達的情感，Skill 會根據這些資訊為您推薦或創造一個獨特且有記憶點的設計方向。

**Q4: 生成的程式碼會是響應式 (Responsive) 的嗎？**
A4: 是的，除非另有指定，生成的程式碼通常會考慮響應式設計，以確保在不同裝置上都有良好的顯示效果。

**Q5: 我可以要求特定的字體、顏色或動畫效果嗎？**
A5: 可以，提供具體的偏好有助於 Skill 更精確地滿足您的需求。Skill 也會在這些基礎上進行創意發揮，以確保整體美學的連貫性與獨特性。

**Q6: 這個 Skill 可以協助優化現有的程式碼或設計嗎？**
A6: 可以，如果您提供現有的程式碼或設計需求，Skill 可以針對其美學、結構或互動性提供改進建議或重構方案，使其更具獨特風格。
