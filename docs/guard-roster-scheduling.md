# 前警守衛排班系統

# 功能說明

自動化處理東陽實業廠的守衛排班表，執行稽核、修正、標記，並產生系統所需的匯入檔案。

# 解決什麼問題

本技能旨在解決東陽實業廠守衛排班作業中，人工排班易出錯、稽核費時、以及需手動轉換為系統匯入檔的痛點。透過自動化檢查與生成，確保排班合規性，提高效率並減少人為錯誤。

# 適合什麼情境

*   每月排班完成後需要檢查合規性時。
*   排班表有違規事項，需要自動標記或修正時。
*   需要將 Excel 排班表轉換為 901907 或 012 系統匯入檔時。
*   管理者想快速了解排班表的詳細情況和潛在問題時。
*   新進人員需要快速了解排班規則時。

# 注意事項

*   **檔案路徑：** 所有相關檔案（排班表、行事曆、匯入檔）預設存放於 `e:/2026andigravity2/01-工作專案/排班系統/`。請確保檔案存在且路徑正確。
*   **規則依據：** 排班稽核與產生檔案的規則基於預設的排班規則定義（包含人員輪替、班別時數、連勤上限、例假日排班等）。
*   **Excel 格式：** 排班表 Excel 需維持固定格式（如內容中的「解析 Excel 結構」所述），格式變動可能導致判讀錯誤。

*   **行事曆更新：** 確保 `2026年東陽行事曆_copy.pdf` 為最新版本，以正確判斷假日。若年份有變動，請同時更新檔案名稱或技能設定。
*   **技術限制：** 部分修正與格式化功能依賴 PowerShell + Excel COM 物件，需確保執行環境支援。

# 常見問題

**Q1: 如果排班表格式跑掉怎麼辦？**
A1: 本技能依賴固定的 Excel 格式來讀取資料。若排班表格式變動，技能可能無法正確讀取或產生錯誤。請確認排班表符合標準結構，例如每位人員的班別/時數資訊應在固定行。

**Q2: 技能可以自動修改所有違規嗎？**
A2: 目前技能主要提供詳細的「稽核」報告，並能自動對「後警上班」的儲存格進行灰色標記。對於更複雜的排班邏輯違規，技能會清楚列出違規項目，但仍需人工判斷後手動調整排班表，以確保符合最佳實踐。

**Q3: 為什麼產生的系統匯入檔是 `.xls` 格式而不是 `.xlsx`？**
A3: 這是根據後端系統的相容性要求，生成 `.xls` 格式以確保能順利匯入。

**Q4: 我可以自訂排班規則嗎？**
A4: 排班規則目前定義在技能的內部邏輯中。若需修改或調整，例如輪班循環、連勤上限等，需要專業人員調整技能底層設定。

**Q5: 如果東陽行事曆更新了，我需要做什麼？**
A5: 請將最新的行事曆 PDF 檔案更新至 `e:/2026andigravity2/01-工作專案/排班系統/` 路徑下，並確保檔案名稱與技能預期的 `2026年東陽行事曆_copy.pdf` 一致，以便技能能正確判斷假日。
