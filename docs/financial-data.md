# 财务数据获取与交叉验证规范

# 功能說明

获取企业财务数据并进行多来源交叉验证，确保数据准确性与可靠性，并明确标注数据差异及其原因。

# 解決什麼問題

在金融研究中，企业财务数据来源复杂多样，不同平台的数据可能存在不一致的情况。本 Skill 旨在解决数据来源不确定、数据质量参差不齐、缺乏有效验证机制的问题，通过强制要求从两个独立来源获取并比较数据，大幅提升财务数据的准确性和可信度，避免因数据错误导致的分析偏差。

# 適合什麼情境

*   需要获取特定上市公司（美股、港股、A股）的年度或季度财务报表数据（如收入、净利润、毛利率、经营现金流、资产负债率等）。
*   进行公司估值分析或撰写金融研究报告时，对关键财务指标的准确性有高要求。
*   在多个数据平台获取到冲突的财务数据，需要进行快速核实和解释差异。
*   希望自动获取并标准化呈现带有来源和验证结果的财务数据。

# 輸出內容

# 注意事項

*   **强制双来源验证**：每个关键财务数据必须从两个独立来源获取，并进行比较。
*   **误差处理机制**：误差超过 1% 的数据将强制进行标记。对于大于 5% 的重大差异，Skill 将提示用户必须查阅原始财报核实，而非直接提供可能错误的数值。
*   **未上市公司**：如果请求的是未上市公司数据，且只有一个一手数据来源时，数据前会标记 `[估计]`，且不执行交叉验证。用户需自行判断数据的可靠性。
*   **季度 vs 年度数据**：Skill 优先使用年度数据进行交叉验证。季度数据在某些来源可能存在更新滞后。
*   **原始财报优先**：如果通过两个平台获取的数据均与公司的官方原始财报（如 SEC 10-K/10-Q、HKEX 年报 PDF、巨潮资讯 PDF 等）不符，以原始财报为准，并会标记来源可能错误。
*   **自动来源选择**：Skill 会根据股票的类型（美股、港股、A股）和预设的优先级自动选择数据源。

# 常見問題

**Q: 为什么获取的数据会有差异？**
A: 财务数据差异是常见的。主要原因包括：
    *   **会计准则不同**：最常见的是 GAAP（通用会计准则）与 Non-GAAP（非通用会计准则）的差异，尤其体现在利润类数据上。
    *   **汇率换算**：不同数据源在进行港币/人民币/美元换算时，可能采用不同时间点的汇率，导致数值差异。
    *   **财年定义**：部分公司并非按照自然年（1月1日-12月31日）定义财年（例如苹果的财年通常在9月或10月结束）。
    *   **合并口径**：是否包含少数股东权益等合并范围的差异。
    *   **数据更新滞后**：部分数据平台可能尚未及时更新最新的财报数据。
本 Skill 会尽量标注这些差异的可能原因。

**Q: 如果数据差异很大（超过5%），我该怎么办？**
A: 当 Skill 标记为 `❌` 并提示"数据存在重大差异"时，意味着数据可靠性存疑。用户应**立即停止**直接使用该数据，并**必须**手动或通过其他工具查阅公司的官方原始财报（如 SEC EDGAR 上的 10-K/10-Q 文件、HKEX 披露易的年报 PDF 或巨潮资讯的 PDF）进行核实，以原始财报为最终依据。

**Q: 这个 Skill 能获取未上市公司的财务数据吗？**
A: 可以。但由于未上市公司缺乏公开透明的财务数据来源，通常只有一个一手数据来源。在这种情况下，Skill 会在数据前标记 `[估计]`，且不会执行交叉验证。用户需自行判断数据的可靠性。

**Q: Skill 会自动选择数据源吗？**
A: 是的，Skill 会根据你请求的股票和市场，自动按照预设的优先级列表（如美股的 macrotrends/stockanalysis，港股的 aastocks/macrotrends，A股的东方财富/巨潮资讯）进行数据获取和验证。

**Q: 我可以指定只使用某个数据源吗？**
A: 本 Skill 的核心价值在于强制进行双源交叉验证，以提高数据准确性。因此，它目前不直接支持仅指定一个数据源。如果用户对特定来源有高度偏好，建议在请求中明确提及，但 Skill 仍会尝试进行验证。最终，原始财报是最高优先级的来源。
