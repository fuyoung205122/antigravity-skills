# Skill 名稱

algorithmic-art (演算法藝術生成器)

# 功能說明

結合計算美學哲學、種子隨機性與互動式 p5.js 程式碼，自動生成具備大師級工藝感、可互動微調的生成藝術（Generative Art）單一網頁。

# 解決什麼問題

1. **美學靈魂的缺失**：傳統生成藝術程式碼往往流於隨機幾何圖形堆疊。本 Skill 透過先建構「演算法哲學（Algorithmic Philosophy）」與「概念種子（Conceptual Seed）」，賦予程式碼藝術靈魂與深刻的視覺層次。
2. **無法重複再現（Reproducibility）**：亂數生成的作品往往無法重現。本 Skill 強制使用「種子隨機（Seeded Randomness）」，確保每一個特定的隨機種子都能完美重現相同的視覺傑作。
3. **缺乏直覺的互動手段**：使用者難以直接在程式碼中調整美學參數。本 Skill 透過內建「參數滑桿與控制面板」，讓藝術家與使用者能在瀏覽器中即時探索演算法的各種變化邊界。
4. **版權合規問題**：避免直接複製現有藝術家的作品，改以原創的數學模型、物理流場與混沌理論，生成獨一無二的數位藝術。

# 適合什麼情境

* 當使用者想要用程式碼、數學公式或演算法創作藝術、生成藝術時。
* 需要展示複雜的粒子系統、向量流場、隨機雜訊、自相似碎形或幾何圖樣。
* 需要一個功能完整的生成藝術展示網頁（含種子切換、參數微調、高解析度 PNG 匯出功能）。
* 數位藝術展演、創意程式設計（Creative Coding）學習、或是尋求視覺設計靈感。

# 如何觸發

範例1：
> 「幫我用 p5.js 寫一個流場（Flow Field）的生成藝術，要有風吹過麥田的有機質感。」

範例2：
> 「我想看一個以『量子力學與隨機糾纏』為概念的互動演算法藝術，能用粒子系統呈現嗎？」

範例3：
> 「請為我設計一個名為『有機混亂（Organic Turbulence）』的藝術運動，並提供可調整參數的互動網頁。」

範例4：
> 「使用數學公式生成一個具備自相似性（Self-similarity）的幾何碎形藝術，要能調整遞迴深度。」

範例5：
> 「我需要一個能在瀏覽器中調整粒子數量、速度和色彩，且能透過種子生成 100 種變化的動態生成藝術。」

# 輸入內容

* **核心主題/風格/概念**：如「宇宙塵埃」、「混沌理論」、「侘寂美學」或特定的物理、數學現象。
* **技術要求（選填）**：特定的庫（p5.js）、偏好的演算法（如 Perlin Noise、Voronoi 圖、L-System）或互動滑桿需求。

# 輸出內容

1. **演算法美學哲學書 (.md)**：包含精心命名的藝術運動名稱，與 4-6 段詩意且專業的計算美學宣言，詳述該演算法如何利用雜訊、力學或幾何展現大師級工藝感。
2. **概念種子解讀**：發掘使用者原始需求中，更深層、幽微且高雅的藝術隱喻。
3. **單一 HTML 互動式 Artifact (.html)**：基於 `templates/viewer.html` 規格建立的單一網頁文件，包含：
   * **左側固定邊欄**：包含 Anthropic 品牌視覺、種子導航（前一個、後一個、隨機、指定種子）、自訂演算法滑桿/參數、顏色選擇器與 Actions 區（重設、重新生成、下載圖片）。
   * **右側畫布區**：高畫質、自適應的 p5.js 動態繪圖區域。

# 注意事項

* **嚴格遵循範本**：必須先讀取 `templates/viewer.html`，以此作為**絕對的起點**，保留其固定的 HTML 結構、Anthropic 品牌字體（Poppins/Lora）及版面配置。僅能修改 marked 的 variable 部分（即 p5.js 演算法邏輯與對應的自訂參數滑桿）。
* **種子一致性**：`setup()` 與 `draw()` 中必須嚴格使用 `randomSeed(seed)` 和 `noiseSeed(seed)`，確保相同的種子永遠產生相同的畫面。
* **拒絕平庸與抄襲**：不直接套用現成藝術家作品，而是透過精密的數學調校（如重力、流向阻尼、色彩權重）呈現極高的視覺完成度，避免雜亂無章的隨機雜訊。

# 常見問題

* **Q1：為什麼重整網頁或切換種子時，畫面沒有變化？**
  * A：請確保您在參數修改或切換種子後，有觸發 `updateParam()` 並調用 p5.js 的 `loop()` 或重新執行初始 setup。所有的隨機數值都必須綁定在 `params.seed` 上。
* **Q2：點擊「Download PNG」會下載到什麼？**
  * A：側邊欄內建的 Download 功能會直接擷取右側 p5.js 畫布當前的畫面，匯出為與畫布解析度一致的高品質 PNG 圖檔。
* **Q3：此 Skill 可以使用 3D 繪圖嗎？**
  * A：可以，p5.js 支援 `WEBGL` 渲染模式。但請確保演算法效率，避免在低配備裝置上產生卡頓。
