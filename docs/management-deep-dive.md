# 管理层纵深研究

# 功能說明

深入分析公司管理层的诚信度、战略眼光、执行能力、资本配置能力以及治理结构。

# 解決什麼問題

傳統的投資研究中，對管理層的評估往往流於表面，僅關注履歷、持股比例等。然而，優秀的投資者深知「買股票就是買人」，管理層的品格和能力是公司長期價值的重要驅動因素。本 Skill 旨在解決僅憑表面資訊難以深入評估管理層的問題，通過挖掘公開資訊，對管理層進行更全面、更深入的分析。

# 適合什麼情境

*   當標準投資研究中，管理層評分為★★★或以下時，需要進行深入研究。
*   當管理層是公司核心投資邏輯時。
*   在投資決策前，需要對潛在投資標的管理層的品格和能力有深入了解時。
*   為了更好地理解公司長期價值和潛在風險時。

# 如何觸發

範例1：
`管理层纵深研究 美团`

範例2：
`管理层纵深研究 王兴 美团`

範例3：
`管理层纵深研究 黄仁勋 英伟达`

範例4：
`深入分析 腾讯 马化腾`

範例5：
`评估 苹果 库克`

# 輸入內容

*   **公司名**：指需要研究的公司名稱，例如「美團」、「英偉達」。
*   **人名（可選）+ 公司名**：指在研究特定公司時，需要重點關注的特定管理層人物及其所屬公司，例如「王興 美團」、「黃仁勳 英偉達」。

# 輸出內容

一份詳細的管理層評估報告，包含以下幾個部分：

1.  **關鍵人物速覽**：列出公司關鍵管理層（CEO/董事長、CFO、創始人、實際控制人等）的姓名、任期、背景、持股/期權資訊。
2.  **誠信度評估**：
    *   承諾兌現率分析（承諾內容、場合、兌現情況及評價）。
    *   困難時期表現分析（危機事件、管理層反應、事後評價）。
    *   對利益相關方（股東、員工、客戶、商家、監管等）的態度分析。
3.  **能力評估**：
    *   戰略眼光評估（CEO過去的判斷與實際結果的對比，預測準確度）。
    *   執行能力評估（戰略到落地、組織能力、危機處理、迭代速度）。
    *   資本配置記錄（對重大資本配置決策如併購、回購、分紅、新業務投資的評估，包括戰略邏輯、回報、評分）。
4.  **治理結構評估**：
    *   股權結構風險分析（AB股、創始人持股、VIE結構、獨立董事、大股東變動）。
    *   薪酬合理性評估（高管薪酬與公司利潤/同行對比）。
    *   關聯交易評估（交易對象、內容、金額、公允性）。
5.  **側面驗證**：
    *   員工視角（從公開評價中分析企業文化、管理層評價、工作強度等）。
    *   客戶/商家視角（從公開評價中分析產品滿意度、客戶服務、供應商關係等）。
    *   行業口碑（從業內評價中了解管理層聲譽）。
6.  **綜合評分與結論**：
    *   對誠信度、戰略與執行能力、資本配置能力、治理結構進行評分（1-5分），並給出加權後的綜合評分。
    *   引用段永平的「買人」標準（是否正直、是否有能力、是否願意託付10年）進行總結。
    *   將報告保存為 `reports/{公司名}-management-{YYYYMMDD}.md` 檔案。

# 注意事項

*   本 Skill 的輸出內容高度依賴於公開可獲取的資訊，資訊的完整性和準確性會影響評估結果。
*   部分側麵驗證資訊（如員工評價、客戶反饋）可能僅能獲取摘要或趨勢，具體細節需用戶自行進一步查找。
*   對於尚未公開的敏感財務數據或內部決策，本 Skill 無法獲取。
*   AI 無法進行實際的「吃飯聊天」和面對面訪談，僅能通過公開資訊進行推斷。
*   「誠信度」是評估的關鍵，若發現嚴重問題，應視為一票否決項。

# 常見問題

**Q1: 如果我只提供公司名，是否也能獲得管理層評估？**
A1: 是的，如果只提供公司名，Skill 會自動識別並研究該公司的關鍵管理層，但如果提供具體人名，則會更聚焦於該個人。

**Q2: 報告中的評分標準是什麼？**
A2: 評分標準主要參考了巴菲特和段永平的投資理念，以及資本配置的實際回報。誠信度是最高權重，其次是戰略執行和資本配置能力，治理結構為輔。

**Q3: 報告中提到「AI無法和管理層吃飯」，這會影響評估的準確性嗎？**
A3: 這確實是一個限制。AI 只能基於公開資訊進行推斷，無法獲得第一手的、非公開的資訊。因此，報告的評估是基於現有資訊的最佳推論，但可能不如親身接觸來得全面。

**Q4: 為什麼報告會保存到本地檔案？**
A4: 這是為了方便用戶長期保存和查閱詳細的研究報告。使用者可以通過指定的檔案路徑找到報告。

**Q5: 我發現報告中的某些資訊不準確，可以如何處理？**
A5: Skill 的資訊來源於公開數據，可能會存在延遲或不準確的情況。建議用戶將報告作為初步參考，並結合自己的研究進行驗證和修正。
