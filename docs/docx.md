# Skill 名稱

DOCX 文件處理工具 (docx)

# 功能說明

此 Skill 用於建立、閱讀、編輯或操作 Word 文件 (.docx 檔案)。

# 解決什麼問題

自動化 Word 文件 (.docx) 的建立、修改與分析，無需手動操作，提升文件處理效率和一致性。它能處理複雜的格式需求、追蹤修訂和內容重組，大幅簡化專業文件製作流程。

*   自動產生報告、備忘錄、信函、範本等專業文件，包含目錄、標題、頁碼、抬頭等標準格式。
*   批量修改現有 Word 文件，例如執行文字查找替換、插入或替換圖片。
*   從 Word 文件中提取內容、重新組織資訊或轉換為其他格式（如 Markdown）。
*   處理文件中的追蹤修訂與註解，或將內容轉換為格式精美的 Word 文件。
*   需要程式化控制 Word 文件內容和結構的任何情境。

# 注意事項

*   **不適用範圍**：本工具不處理 PDF、試算表、Google Docs 或與文件產生無關的一般程式碼任務。
*   **頁面設定**：使用 `docx-js` 建立新文件時，預設為 A4 紙張大小。若需使用美國信紙 (US Letter) 或其他尺寸，請務必在 `Document` 設定中明確指定頁面寬度和高度（例如 US Letter 為 12240 x 15840 DXA）。
*   **表格處理**：
    *   表格寬度 (`Table.width` 和 `TableCell.width`) 必須使用 DXA 單位 (`WidthType.DXA`)，切勿使用 `WidthType.PERCENTAGE`，因為它在 Google Docs 中存在相容性問題。
    *   表格總寬度必須等於所有欄寬度的總和。每個儲存格也需設定其寬度。
    *   儲存格的陰影應使用 `ShadingType.CLEAR`，而非 `SOLID`，以避免產生黑色背景。
*   **清單格式**：切勿直接在文字中插入 Unicode 項目符號（如 `•` 或 `\u2022`）。應使用 `numbering` 設定中的 `LevelFormat.BULLET` 或 `LevelFormat.DECIMAL` 來建立具有正確格式的清單。
*   **追蹤修訂**：編輯現有文件時，應以「Claude」作為修訂作者，除非使用者明確要求使用其他名稱。
*   **特殊字元**：新增內容或編輯 XML 時，請務必使用 XML 實體（如 `&#x2018;` ‘, `&#x2019;` ’, `&#x201C;` “, `&#x201D;` ”）來表示智慧引號和撇號，以確保專業的排版和字元顯示。
*   **文件結構**：直接編輯 XML 時，請務必遵循 Word 的 XML 結構和元素順序（例如 `<w:pPr>` 中的 `<w:pStyle>`, `<w:numPr>`, `<w:spacing>`, `<w:ind>`, `<w:jc>`, `<w:rPr>` 順序），否則可能導致文件損壞。
*   **圖片插入**：插入圖片時，`ImageRun` 物件必須指定 `type` 參數（如 `"png"`, `"jpg"`）。
*   **分頁符號**：`PageBreak` 元素必須包含在 `Paragraph` 元素內部，不能獨立存在。
*   **頁首/頁尾**：請勿使用表格來建立頁首/頁尾的分隔線或多欄佈局，應使用段落邊界或定位點 (`tabStops`)。

# 常見問題

1.  **為什麼我建立的 Word 文件頁面大小不正確？**
    *   **答**：`docx-js` 預設為 A4 紙張大小。若需使用美國信紙 (US Letter) 或其他尺寸，請務必在 `Document` 設定中的 `sections` > `properties` > `page` 參數下明確指定頁面 `width` 和 `height`（例如 US Letter 為 12240 x 15840 DXA）。
2.  **我的表格在 Google Docs 中顯示不正常，該怎麼辦？**
    *   **答**：表格寬度 (`Table.width`) 和每個儲存格的寬度 (`TableCell.width`) 必須使用 DXA 單位 (`WidthType.DXA`)，切勿使用 `WidthType.PERCENTAGE`，因為它與 Google Docs 存在相容性問題。此外，請確保表格的總寬度 (`Table.width`) 嚴格等於所有 `columnWidths` 的總和。
3.  **如何正確地在文件中建立項目清單（子彈符號或編號清單）？**
    *   **答**：切勿直接在文字中插入 Unicode 項目符號（如 `•` 或 `\u2022`）。應使用 `numbering` 配置中的 `LevelFormat.BULLET` (子彈符號) 或 `LevelFormat.DECIMAL` (編號) 來建立具有正確縮排和樣式的清單。
4.  **我在編輯 XML 時，插入的追蹤修訂為何無法正確顯示？**
    *   **答**：追蹤修訂 (Insertion/Deletion) 應替換整個 `<w:r>...</w:r>` 元素，並將 `<w:del>` 和 `<w:ins>` 作為 `<w:r>` 的同級元素。同時，請將原始 `<w:r>` 中的 `<w:rPr>` 格式屬性複製到追蹤修訂的 `<w:r>` 中，以保留粗體、字體大小等原始樣式。
5.  **為何 Word 文件中的智慧引號和撇號在我編輯後變成了普通引號？**
    *   **答**：在編輯 XML 或新增文字內容時，請務必使用 XML 實體（如 `&#x201C;` 對應 “，`&#x201D;` 對應 ”，`&#x2019;` 對應 ’）來表示智慧引號和撇號，以確保在 Word 中顯示專業的排版字元。
6.  **如何讓目錄 (Table of Contents) 正確生成？**
    *   **答**：確保文件中的標題段落 (`Paragraph`) 僅使用 `HeadingLevel` 屬性（如 `HeadingLevel.HEADING_1`），而沒有額外的自定義樣式覆蓋。同時，標題樣式 (`paragraphStyles`) 中必須包含 `outlineLevel` 屬性（H1 為 0，H2 為 1 等），這對於 TOC 的生成是必需的。
