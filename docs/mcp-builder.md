# MCP Server Development Guide

# 功能說明

協助建立高品質的 MCP (Model Context Protocol) 伺服器，讓大型語言模型 (LLM) 能透過精心設計的工具與外部服務互動。

# 解決什麼問題

讓大型語言模型 (LLM) 能夠順利且有效地與各種外部 API 或服務進行互動。本 Skill 提供一套標準化的開發指南和最佳實踐，解決 LLM 整合外部服務時面臨的工具設計、協定理解、實作和評估等挑戰，確保開發出具備高效率和準確性的 MCP 伺服器。

# 適合什麼情境

*   整合外部 API 或服務到 LLM 應用中。
*   從零開始開發新的 MCP 伺服器。
*   使用 Python (FastMCP) 或 Node/TypeScript (MCP SDK) 建立 MCP 服務。
*   希望提高 LLM 使用工具的效率、準確性和可靠性。
*   需要評估和測試 MCP 伺服器的效能，確保其能處理複雜的真實世界任務。
*   尋求 MCP 協議、工具設計或實作的最佳實踐。

# 注意事項

*   本 Skill 專為提供 MCP 伺服器開發的「指南」和「最佳實踐」，而非直接執行建置工作。
*   內容涵蓋 Python (FastMCP) 和 Node/TypeScript (MCP SDK) 兩種主要開發語言，並推薦使用 TypeScript。
*   部分資訊會指示使用者查閱外部連結 (例如 MCP 協議官方網站、GitHub SDK README) 或內部的參考文件 (`./reference/*.md`)。使用者可能需要自行存取這些資源以獲取完整資訊。
*   伺服器設計著重於讓 LLM 能夠有效利用工具，因此工具命名、上下文管理、錯誤訊息等設計原則至關重要。
*   評估階段需要創建特定 XML 格式的問題集，並遵循嚴格的設計準則。

# 常見問題

*   **Q：這個 Skill 能直接幫我建立 MCP 伺服器嗎？**
    *   A：不能，它提供的是開發指南、最佳實踐和參考資料，協助您自行建立高品質的 MCP 伺服器。
*   **Q：我該如何選擇 MCP 伺服器的開發語言？**
    *   A：指南推薦使用 TypeScript，因其高品質的 SDK 支援、廣泛兼容性及靜態型別等優勢。但本 Skill 也提供了 Python 的相關指南。
*   **Q：提到的外部文件 (例如 GitHub README) 我可以直接取得內容嗎？**
    *   A：本 Skill 會提供取得這些文件的路徑或方法 (例如 WebFetch 指令)，你可能需要使用平台的特定工具來實際獲取其內容。
*   **Q：什麼是 MCP (Model Context Protocol)？**
    *   A：MCP 是一種協議，旨在讓 LLM 能夠以標準化、高效的方式，透過精心設計的工具與外部服務進行互動。
*   **Q：如何確保我的 MCP 工具設計良好？**
    *   A：遵循指南中的「深度研究與規劃」階段，特別注意清晰、描述性的工具命名、有效的上下文管理、可操作的錯誤訊息，以及明確的輸入/輸出 Schema 定義。
*   **Q：我應該如何為我的 MCP 伺服器進行測試？**
    *   A：在「審查與測試」階段，建議執行代碼品質檢查，並使用 MCP Inspector 等工具進行測試。在「建立評估」階段，則需創建綜合性的評估問題集來測試 LLM 使用工具的有效性。
