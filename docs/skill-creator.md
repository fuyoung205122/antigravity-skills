# Skill Creator

# 功能說明

一個用於從零開始建立、疊代優化、測試評估以及自動打包新 Skill 的全方位開發工具。

# 解決什麼問題

在開發 AI Skill 時，手動編寫指令（Prompt Engineering）往往難以掌握準確度，且缺乏量化的測試方法。Skill Creator 解決了 Skill 開發流程中「定義不明、測試不嚴謹、觸發不穩定、缺乏對比數據」等痛點，讓 Skill 開發變成一個可觀測、可優化的科學循環。

# 適合什麼情境

*   **從無到有建立 Skill**：當你有一套固定的工作流程（例如：特定的數據轉換、代碼生成風格），想將其固化為 Skill。
*   **優化既有 Skill**：現有的 Skill 表現不穩定、經常漏掉細節，需要透過測試與反饋來改進。
*   **觸發準確度提升**：當 Skill 在該出現時不出現，或不該出現時亂觸發，需要優化其 Metadata 描述。
*   **效能評估與對比**：想要知道新版本的 Skill 指令是否真的比舊版本（或沒用 Skill 時）效果更好。
*   **自動化打包**：完成開發後，需要快速產出可供安裝的 `.skill` 檔案。

# 如何觸發

範例1：
「我想建立一個新的 Skill，專門用來把雜亂的會議記錄轉換成標準的 Notion 頁面格式。」

範例2：
「幫我優化這個名為 `data-cleaner` 的 Skill，它在處理大型 CSV 時常常出錯。」

範例3：
「幫我為這個 Skill 撰寫 5 個測試案例，並對比它在有無使用 Skill 情況下的輸出差異。」

範例4：
「執行這個 Skill 的 Benchmark 測試，我想看看它的 token 使用量和平均耗時。」

範例5：
「我的 Skill 觸發不太準確，請幫我生成 20 個測試 Query 並優化它的描述字段。」

# 輸入內容

*   **意圖描述**：Skill 的用途、觸發時機、預期輸出格式。
*   **測試資源**：測試用的 Prompt（測項）、參考文檔或範例檔案。
*   **評估標準**：如何判定輸出是成功的（例如：必須包含摘要、格式必須為 JSON 等）。
*   **既有文件**（若是優化）：現有的 `SKILL.md` 或相關腳本。

# 輸出內容

*   **SKILL.md**：包含 YAML 前言（名稱、描述）與詳細指令的 Skill 核心文件。
*   **測試報告 (Eval Viewer)**：一個互動式網頁，供使用者逐項審閱測試結果並留下回饋。
*   **數據指標 (Benchmark)**：包含成功率（Pass Rate）、處理時間、Token 消耗量等量化數據。
*   **優化建議**：針對觸發描述的 A/B 測試結果與最佳建議。
*   **打包檔案**：最終可分發安裝的 `.skill` 檔案。

# 注意事項

*   **安全性限制**：禁止建立包含惡意代碼、木馬或具誤導性、用於未授權數據抓取的 Skill。
*   **環境依賴**：在某些受限環境（如 Cowork）下，無法自動開啟瀏覽器，需手動開啟產出的靜態 HTML 文件。
*   **併行限制**：運行大規模測試時，請注意子代理（Subagents）的額度與系統資源。

# 常見問題

**Q：Skill Creator 可以幫我寫腳本（Python/Bash）嗎？**
A：可以。如果測試過程中發現某些步驟重複性高且邏輯固定，它會建議並協助你將其編寫為 Skill 內部的 `scripts/`。

**Q：如果不滿意測試結果怎麼辦？**
A：這是一個循環流程。你可以直接在 Eval Viewer 留下意見，Skill Creator 會讀取 feedback.json 並自動調整指令進入下一輪測試（Iteration）。

**Q：為什麼需要 Baseline（基準測試）？**
A：為了確保你的 Skill 真的有價值，我們會對比「使用 Skill」與「完全不使用 Skill（或用舊版）」的差異。
