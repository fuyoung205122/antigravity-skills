# AI Berkshire 投资研究 Skill

# 功能說明

利用巴菲特、芒格、段永平、李录四位投资大师的综合分析框架，对指定公司进行系统化的深度投资研究。

# 解決什麼問題

AI在進行投資研究時，容易出現數據偏差、信息不完整、分析片面等問題。此 Skill 旨在透過結構化的分析框架、嚴謹的數據驗證流程以及對投資大師思想的模擬，產出更為全面、客觀、具有洞察力的投資研究報告，幫助用戶規避 AI 研究的潛在陷阱。

# 適合什麼情境

*   **進行股票投資前的深度研究**：用戶想深入了解一家公司的基本面、商業模式、護城河、管理層、估值及潛在風險。
*   **驗證現有投資判斷**：用戶已有對某家公司的初步看法，希望透過專業框架進行二次驗證。
*   **學習投資大師的分析方法**：用戶希望學習巴菲特、芒格、段永平、李录的投資理念和分析邏輯。
*   **獲取結構化的行業與公司分析報告**：需要一份標準化的、數據驅動的投資研究報告。

# 如何觸發

範例1：
請對 "特斯拉" 進行投資研究，分析其商業模式、護城河、管理層、估值，並對比巴菲特、芒格、段永平、李录的觀點。

範例2：
使用 AI Berkshire 投資研究 Skill，對 "貴州茅台" 進行分析，聚焦其商業本質和品牌價值。

範例3：
我想了解 "騰訊控股" 的風險因素和最大潛在風險，請用芒格的反向思考法進行分析。

範例4：
請對 "英偉達" 進行估值分析，並結合李录的文明演进框架，預測其未來趨勢。

範例5：
對 "蘋果公司" 進行全面投資研究，特別是其護城河的持久性和管理層的資本配置能力。

# 輸入內容

*   **公司名稱**：進行投資研究的目標公司名稱。
*   **（可選）研究重點**：用戶可以指定希望重點分析的方面，例如：商業模式、護城河、管理層、估值、風險等。

# 輸出內容

一份結構化的投資研究報告，內容包含：

*   **報告開頭**：
    *   公司名稱
    *   信息豐富度評級（A/B/C）
    *   AI 研究局限性聲明
*   **研究框架各模塊的詳細分析**：
    *   **前置步驟：AI 研究偏見自覺**：信息豐富度評級及應對策略，C 級公司的第一性原理研究法，偏見自查清單。
    *   **第一步：數據收集**：收入結構、財務指標、競爭格局、商業模式與護城河、技術能力、管理層、行業前景、風險因素、當前估值、多空雙方核心論點。
    *   **數據交叉驗證記錄**：關鍵數據的程式化驗證結果。
    *   **第二步：生意本質分析**（段永平）：生意本質、收入結構、盈利能力趨勢、商業模式、生態粘性、毛利率、經營槓桿，段永平式追問。
    *   **第三步：護城河評估**（巴菲特）：五類護城河的驗證，護城河趨勢，巴菲特式追問。
    *   **第四步：逆向思考與風險清單**（芒格）：公司失敗路徑、歷史類比、跨學科分析、偏誤自查、空方論點，芒格式追問。
    *   **第五步：管理層評估**（段永平+巴菲特）：關鍵決策復盤、資本配置能力、股東利益一致性、組織能力、企業文化，段永平式追問。
    *   **第六步：行業與文明趨勢**（李录）：文明級範式轉移判斷、歷史技術革命類比、TAM 分析、產業鏈位置、技術路線風險、客戶/供應商集中度，李录式追問。
    *   **第七步：估值與安全邊際**（巴菲特+段永平）：當前市場定價、反向 DCF、三情景估值（精確計算），與歷史/同行對比，段永平式追問。
    *   **第八步：綜合決策備忘錄**：各維度結論匯總，最終決策表格（空倉者、持倉者、賣出信號、加倉信號），四位大師的模擬點評。
*   **報告結尾**：
    *   AI 分析置信度與實際投資確定性的區分。
    *   （若為 C 級公司）需要一手驗證的問題清單。

**輸出格式**：Markdown 文件，儲存於 `~/[公司名]投資研究報告.md`。

# 注意事項

*   **數據來源規範**：所有財務數據必須來源於規範的獨立第三方平台，並進行交叉驗證。
*   **嚴謹的數據驗證**：涉及關鍵財務數據（如市值、營收、淨利潤、現金儲備、估值指標等）必須通過 `financial_rigor.py` 等工具進行程序化驗證。
*   **AI 研究局限性**：報告將明確標示 AI 研究的局限性，特別是對於信息稀缺（C 級）的公司，會強調 AI 推理的置信度，並建議用戶進行一手調研。
*   **禁止 LLM 心算**：所有複雜計算必須依賴工具完成，嚴禁 AI 進行心算。
*   **研究流程強制性**：AI 研究偏見自覺、數據交叉驗證、關鍵數據工具驗算、報告數據抽檢等步驟是強制性的。
*   **資訊豐富度等級影響**：若公司被評為 C 級（信息稀缺），報告的深度和確定性會受到影響，並強調需要額外的人工驗證。

# 常見問題

**Q1：這個 Skill 的研究報告有多可靠？**
A1：本 Skill 採用結構化的分析框架，並嚴格執行數據交叉驗證和工具驗算，力求提供客觀、有數據支持的分析。但 AI 的研究終究受限於公開數據的質量和廣度，特別是對於信息稀缺的公司，仍建議用戶結合自身判斷和一手調研。報告中會明確標示 AI 分析的置信度。

**Q2：如果我想研究一家未上市的公司，這個 Skill 可以用嗎？**
A2：此 Skill 主要依賴公開的上市公司財務數據和市場信息進行分析。對於未上市公司，由於缺乏公開數據，其研究效果將大打折扣，甚至無法進行。

**Q3：我如何知道報告中的數據是否準確？**
A3：報告中所有關鍵財務數據都經過了嚴謹的交叉驗證和工具驗算，並會標示數據來源。如果在報告中發現您認為有誤的數據，請對照報告中的來源信息進行核實，或向 Skill 反饋。

**Q4：報告中提到的“AI 研究偏見”是什麼意思？**
A4：AI 在分析數據時，可能會因為數據量、數據質量、訓練樣本的傾向性等原因產生偏見。例如，對於信息豐富的公司，AI 可能會傾向於重複市場的共識觀點；對於信息稀缺的公司，AI 可能會因為無法獲取足夠信息而過於保守。此 Skill 強調要對這些潛在偏見保持警惕。

**Q5：如果公司被評為 C 級（信息稀缺），我還能相信這份報告嗎？**
A5：對於 C 級公司，本 Skill 會盡力基於現有信息進行分析，但報告的確定性會降低。您會收到一份「需要一手驗證的問題清單」，這意味著報告中的部分結論是基於有限信息的推導，您需要通過實地考察、產品體驗、與行業內人士交流等方式來補充 AI 的盲區，驗證報告中的關鍵假設。
