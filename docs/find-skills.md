# Skill 名稱

Find Skills (尋找技能)

# 功能說明

協助使用者探索並安裝開源代理技能生態系統中的技能，以擴展其代理程式功能。

# 解決什麼問題

當使用者需要特定功能，但代理程式本身不具備時，此 Skill 能夠幫助他們從龐大的開源技能生態系統中尋找、評估並安裝所需的技能，有效擴展代理程式的能力。

# 適合什麼情境

*   使用者詢問「我該如何做 X」或「有沒有一個技能可以做 X」。
*   使用者想尋找特定的工具、模板或工作流程。
*   使用者表示希望在特定領域（如設計、測試、部署）獲得協助。
*   使用者希望探索並擴展代理程式的既有功能。
*   使用者對提升代理程式能力有興趣。

# 如何觸發

提供至少 5 個實際範例。

範例1：我該如何快速建立一個 React 應用程式？
範例2：有沒有一個技能可以幫我處理前端設計？
範例3：我想找一個可以協助我自動化部署的工具。
範例4：我需要寫單元測試，有什麼相關的技能嗎？
範例5：我的程式碼品質不太好，能幫我找找關於程式碼審查或最佳實踐的技能嗎？
範例6：代理程式，你能幫我找一個處理資料分析的技能嗎？

# 輸入內容

使用者以自然語言描述其所需的功能、任務或特定領域，例如：
*   一個問題或需求描述 (e.g., "how do I do X", "find a skill for X")
*   查詢關鍵字 (e.g., "react performance", "pr review", "changelog")

# 輸出內容

*   找到的相關技能列表。
*   每個技能的詳細資訊，包括：
    *   技能名稱和功能摘要。
    *   安裝次數和來源。
    *   提供安裝指令 (`npx skills add <package>`)。
    *   連結至 skills.sh 供使用者進一步了解。
*   如果沒有找到相關技能，則會告知使用者，並提供替代方案（例如：直接使用代理程式的通用能力協助，或建議使用者自行建立技能）。

# 注意事項

*   **不應僅憑搜尋結果直接推薦技能**：在推薦前，務必驗證技能的品質。
*   **技能品質評估準則**：
    *   **安裝次數**：優先選擇安裝次數超過 1K 的技能。對於安裝次數低於 100 的技能應謹慎。
    *   **來源聲譽**：官方來源 (例如 `vercel-labs`, `anthropics`, `microsoft`) 通常更值得信賴。
    *   **GitHub 星級**：檢查技能源碼儲存庫的 GitHub 星級。星級低於 100 的儲存庫所提供的技能應持懷疑態度。
*   若搜尋結果顯示沒有相關技能，Skill 會主動告知，並可能建議使用者直接使用通用能力或自行開發技能。

# 常見問題

**1. 什麼是 Skills CLI？**
Skills CLI (Command Line Interface)，即 `npx skills`，是開源代理技能生態系統的套件管理器。它提供指令來搜尋 (`npx skills find`)、安裝 (`npx skills add`)、檢查更新 (`npx skills check`) 和更新 (`npx skills update`) 技能。

**2. 如何有效搜尋我需要的技能？**
為提升搜尋效率：
*   使用具體和精確的關鍵字，例如「react testing」優於單純的「testing」。
*   嘗試使用不同的或替代的術語，例如「deploy」若無結果，可嘗試「deployment」或「ci-cd」。
*   留意來自熱門或官方來源的技能，如 `vercel-labs/agent-skills`。

**3. 如果找不到符合我需求的技能怎麼辦？**
如果此 Skill 無法找到相關技能，它會明確告知您，並可能提供兩種選擇：
*   讓代理程式直接利用其通用能力協助您完成任務。
*   建議您考慮使用 `npx skills init` 指令自行建立一個新的技能。

**4. 我該如何評估一個技能的可靠性或品質？**
評估技能品質時，請注意其安裝次數（1K+ 較佳）、來源信譽（官方或知名來源優先）以及其 GitHub 專案的星級（100+ 較佳）。切勿僅根據搜尋結果就安裝或推薦。

**5. 找到技能後，我該如何安裝它？**
當此 Skill 推薦一個技能時，它會提供詳細的安裝指令，例如 `npx skills add vercel-labs/agent-skills@react-best-practices`。您通常可以使用 `-g` 旗標進行全域安裝，並使用 `-y` 旗標跳過確認提示。
